Package io.spine.query

Class EntityColumn

  • All Implemented Interfaces:
    io.spine.query.Column , java.io.Serializable 
    @Immutable() 
public final class EntityColumn<S extends EntityState<out Object>, V>
extends RecordColumn<S, V>

    A queryable column of an entity which can be passed to the query filters.

    Normally, instances of this class are provided by the Spine-generated column enumeration and should not be constructed in the user code directly.

    For each message which represents an entity state and has columns, the Spine routines will generate a nested Column class which exposes all columns of the entity as EntityColumn instances. Example: 

    // Given a message declaration.
message ProjectDetails {
    option (entity).kind = PROJECTION;

    ProjectId id = 1;
    ProjectName name = 2 [(column) = true];
    int32 task_count = 3 [(column) = true];
}

// The following Java class will be generated.
public final class ProjectDetails ... {

    // ...

    public static final class Column {

        private Column() {
            // Prevent instantiation.
        }

        // Returns the "name" column.
       public static EntityColumn<ProjectDetails, ProjectName> name() {...}

        // Returns the "task_count" column.
       public static EntityColumn<ProjectDetails, Integer> taskCount() {...}

        // Returns all the column definitions for this type.
       public static Set<EntityColumn<ProjectDetails, ?>> definitions() {
           Set<EntityColumn<ProjectDetails, ?>> result = new HashSet<>();
            result.add(name());
            result.add(taskCount());
            return result;
       }
    }
}

    The values retrieved via static methods of the Column type may then be passed to a client to form a query request.

    See the Spine code generation routines in tool-base for extensive details on how the types are generated.
