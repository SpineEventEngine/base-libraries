Package io.spine.base

Class Identifier

  • All Implemented Interfaces:
    
public final class Identifier<I>

    Wrapper of an identifier value.

    The following types of IDs are supported:

    • String
    • Long
    • Integer
    • A class implementing Message.

    Consider using Message-based IDs if you want to have typed IDs in your code, and/or if you need to have IDs with some structure inside.

    Here are the examples of such structural IDs:

    • EAN value used in bar codes
    • ISBN
    • Phone number.

    Most data in Spine-based applications come in the form of Protobuf messages. If the data has an identity, we use the convention of the first message field.

    The "first" field is determined by the order in which fields are declared in the Protobuf message definition (reading order from top to bottom), not by the field number. For example, in the following message: 

    message Example {
    string name = 2;
    string id = 1;
}
    the name field is considered "first" because it appears first in the declaration, even though its field number (2) is greater than id's field number (1).

    This approach provides several benefits:

    • Easier to read and understand — developers do not need to mentally sort fields by their numbers.
    • Supports field deprecation scenarios — if an ID field needs to be replaced (e.g., upgrading from int32 to int64), the new field can be added at the top while the old field is deprecated in place.
