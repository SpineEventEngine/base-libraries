      • either

         abstract B either(Array<Either<B>> parameters)

        Adds a predicate to be treated in disjunction with the existing predicates.

        All expressions passed with every Either parameter are treated with OR behavior.

        Example. 

            ProjectView.query()
               .either(builder-> builder.daysSinceStarted()
                                         .isGreaterThan(30),
                       builder-> builder.status()
                                         .is(DONE))
               .build();

        The ProjectView query above targets the instances which are either started more than thirty days ago, or those which are in DONE status.

        Each Either is a lambda serving to preserve the current QueryBuilder with its API and syntax sugar for creating the new predicates, but in a disjunction context.

        Another example. 

        ImmutableList<Project.Status> statuses = //...
    ProjectView.query()
               .either((builder)-> {
                   for (Project.Status status : statuses) {
                       builder.status().is(status);
                   }
                   return builder;
               }).build();

        This example creates a query for the ProjectView instances which have one of the expected statuses. Note that either(..) is passed with a single argument lambda. Each predicate appended to the builder inside of the passed lambda is treated as a disjunction predicate. Basically, that is just a short form of the expression as follows: 

        ImmutableList<Project.Status> statuses = //...
    ProjectView.query()
               // Performs the same as in the previous example. Much less elegant though.
               .either(builder-> builder.status().is(statuses.get(0)),
                       builder-> builder.status().is(statuses.get(1)),
                       builder-> builder.status().is(statuses.get(2)),
                       //...
                       builder-> builder.status().is(statuses.get(lastOne)))
               .build();

        If several Either lambdas are passed to the either(..), all predicates appended to the builder in them are treated together in an OR fashion.

        You may extract lambdas into variables to simplify the code even further: 

        Either<ProjectView.QueryBuilder> startedMoreThanMonthAgo =
                    project-> project.daysSinceStarted()
                                      .isGreaterThan(daysSinceStarted);
   Either<ProjectView.QueryBuilder> isDone =
                    project-> project.status()
                                      .is(statusValue);
    ProjectView.Query query =
            ProjectView.query()
                       .either(startedMoreThanMonthAgo, isDone)
                       .build();

        this instance of query builder, for chaining

        @CanIgnoreReturnValue() abstract B sortAscendingBy(RecordColumn<R, out Object> column)

        Tells to sort the query results in the ascending order of the values in the specified column.

        Each call to this method adds another sorting directive. Directives are applied one after another, each following determining the order of records remained "equal" after the previous sorting.
