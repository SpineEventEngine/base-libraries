Package io.spine.format

Class Format

  • All Implemented Interfaces:
    
public class Format<T extends Object>

    A format of a file supported by the io.spine.format package.

    This class is an abstract base for the supported format classes. A format of a file can be obtained using the Format.of function.

    You can check if a file is in the supported format using the function File.hasSupportedFormat.

    The following formats handle Protobuf message types:

    • ProtoBinary — storing and parsing in Protobuf binary encoding. This is the performance and memory-efficient format.

    • ProtoJson — handles ProtoJSON format. This format should be used for debugging purposes or for sending data to external systems that do not understand the ProtoBinary format.

    • Json — handles JSON format.

    • Yaml — handles the YAML format.

    Formats offered by this package do not allow mixing regular Java data types with Java code generated by Protobuf. This is so because the ProtobufFactory provided by Jackson is not backed by the official Protobuf library.

    This is why regular Java or Kotlin class instances should be handled using Json and Yaml formats, while Protobuf types created are to be handled by ProtoBinary or ProtoJson formats.
