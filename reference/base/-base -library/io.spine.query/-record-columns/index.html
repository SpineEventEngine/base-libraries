Base Library/io.spine.query/RecordColumns

RecordColumns

@Retention(value = RetentionPolicy.SOURCE)
@Target(value = ElementType.TYPE)
@Documented
annotation class RecordColumns

Annotates the types which define the columns of the Protobuf messages stored as records in a storage.

We recommend following some naming patterns and approaches when designing such types.

Let's review an example: 

    import static io.spine.query.RecordColumn.create;

    // ...

@RecordColumns(ofType = ProjectView.class)
    public static class ProjectViewColumns {

public static final RecordColumn<ProjectView, ProjectName> project_name =
              create("project_view", ProjectName.class, ProjectView::getProjectName);

public static final RecordColumn<ProjectView, UserName> assignee =create("assignee", UserName.class, (p) -> p.getAssignee().getName());
    }

The columns are declared as public static final fields. There are two reasons for that.

  1. It allows to refer to columns via a static import, as it's nice to shorten the expression in which a column declaration is used.
  2. We want to preserve the type of the column value (which is determined by the second generic parameter) in order to put bounds to the compared values in scope of a query.

Here is how it plays together with a RecordQuery: 

    import static com.acme.ProjectViewColumns.project_name;

    // ...

    ProjectName myProjectName = // ...

    // Selects all `ProjectView`s by the project name.
RecordQuery<ProjectId, ProjectView> query =
            RecordQuery.newBuilder(ProjectId.class, ProjectView.class)
                       .where(project_name).is(myProjectName)
                       .build();

Note that myProjectName is checked to be of ProjectName type, at the time of code compilation.

Also, the snake_case notation used in the declaration of project_name field. As many field names contain at least two words, such an approach allows to easily distinguish a field from a Java variable holding a value to which the field values are compared (myProjectName in the example above).
